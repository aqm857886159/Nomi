## 一句话总结

前七轮学界收敛到「单向 DAG 是病、要闭环回灌」（Co-Director → HollyWood Town → MUSE 一路印证）。本轮出现**两个可直接抄进 Nomi 的具体机制**，都不碰模型内部、model-agnostic、能架在 vendor API 上：

1. **GroundShot（2026-06-18，训练-free，Fudan/百度系）** 把「怎么排镜头生成顺序」从 Nomi 现在的**朴素编号排序**升级成**「按实体依赖拓扑排序」**——先生成会被后面反复用到的「参考源镜」（立住主角/关键物体的那几镜），把**校验过可靠的实体证据**存进 entity-level 视觉记忆，再喂给依赖它们的后续镜。这直击 Nomi 命门（跨镜身份），且**正好接在 Nomi 已有的 `dependencyWaves.ts` + `generationReferenceResolver.ts` + 镜级校验/有界重试基建上**，不是从零造。
2. **KathaTrace（2026-07-01，两周新，generator-agnostic 诊断 benchmark + 指标）** 给了「镜间转场语义」这个 DirectorBench 指认的全行业瓶颈**第一把可量化的尺子**（Semantic Trajectory Gap = 文字可推断度 − 画面可推断度，量「视觉化时丢了多少转场含义」），还附一个 Semantic Compass 探针做「生成后修复 + storyboard 选优」。这正好补 Nomi eval 体系在「转场/叙事连贯」这条 lane 的量化缺口。

**本轮最该动的一件事：把 Nomi 拆镜头规划师的镜头生成顺序，从 `storyboardPlan.ts:285` 现在的朴素 `a.index - b.index` 排序，升级成 GroundShot 式「实体依赖拓扑排序 + 校验过的实体记忆回填参考」——先立住主角镜、验证可靠后再供给后续镜。** 这是把前七轮「闭环回灌」的抽象结论落成一个具体、training-free、能接现有基建的改动。第二件（次优先）：把 KathaTrace 的 STG 指标接进 eval 体系，量 Nomi 拆镜头产物的镜间转场语义损失。诚实边界见各条。

## 🟢 可落地（这月能进 Nomi）

### GroundShot：把「镜头生成顺序」从朴素编号排序改成「实体依赖拓扑排序 + 校验过的实体记忆回填参考」，training-free、model-agnostic（2026-06-18）
[arxiv 2606.20799](https://arxiv.org/abs/2606.20799) · **本轮核实无开源代码 repo**（4 组件管线在正文/摘要写得够全，可照抄逻辑）· 作者 Yixuan Lai / Tianjia Shao / Kun Zhou / Weijia Dou / Siyu Zhu(复旦) / Jingdong Wang(百度系) · 摘要明写 **"training-free, model-agnostic agentic framework … requiring no additional training or model modification"** · 日期：2026-06-18（窗内最新）。

- **它打的正是 Nomi 命门 + 一个 Nomi 现在没做的维度**：跨镜身份不只靠「给每镜塞参考图」，更靠**「先生成哪一镜」**。GroundShot 的洞察：**recurring 实体（反复出现的角色/物体/地点）应该在「立住它的那一镜」先生成，再把该镜里校验可靠的实体外观当参考，喂给后面所有用到它的镜。** Nomi 现在 `storyboardPlan.ts:285` 是 `[...plan.shots].sort((a, b) => a.index - b.index)`——**纯按镜号顺排**，没有「谁该先生成来当别人的参考」这个概念。
- **四组件管线（可分开借鉴，全是编排层不碰模型）**：
  1. **脚本解析**：LLM 读全脚本，抽出全局实体集，给每个实体分类型（character/object/location）+ 跨镜链接同一实体给同一 entity ID + 分优先级（角色 > 物体 > 地点）。
  2. **镜头调度**：从「预测的参考源镜」建**依赖图**，**拓扑排序**成生成顺序（参考源镜排前）。← **这是替换 `storyboardPlan.ts:285` 朴素排序的那块**。
  3. **实体级视觉记忆**：存每个实体的 canonical 参考 + canonical-consistent 补充参考。← 接 `generationReferenceResolver.ts`（160 行）/`referenceSlots.ts`。
  4. **镜头生成**：为当前镜取参考 → 选 T2V 或 Ref2V → **verify & repair** → 从生成结果 ground 出实体证据 → **只把可靠的**更新进记忆。← verify/repair 正好接 Nomi 已有 `gatherShotVerifyInputs.ts` + `storyboardLoopBudget.ts`（有界重试）。
- **落到哪个文件 + 怎么改**：
  - `src/workbench/generationCanvas/agent/storyboardPlan.ts:285` — 把朴素 index 排序换成「实体依赖拓扑排序」：先让 LLM/规划师标出每镜引入的 recurring 实体 + 哪些镜是某实体的「参考源镜」，建依赖图，参考源镜拓扑靠前。
  - `src/workbench/generationCanvas/runner/dependencyWaves.ts`（5080 字节，已存在执行波次排序）— GroundShot 的调度是**规划期**的生成顺序，dependencyWaves 是**执行期**的边依赖波次；两者要对齐（规划产出的顺序须被执行波次尊重），是这块的集成风险点。
  - `src/workbench/generationCanvas/runner/generationReferenceResolver.ts` + `referenceSlots.ts` — entity-level 记忆的「按 entity ID 取回可靠参考」挂这里，和 Nomi 现有「连线在前单源参考」口径（记忆 `reference-ordering-single-source-and-upload-fixes`）对齐，别再开第二套参考真源（违 P1）。
  - `gatherShotVerifyInputs.ts` + `storyboardLoopBudget.ts` — 「生成后 ground 实体证据、只把可靠的入记忆」这步，复用现有镜级 verify + 有界重试，不新造校验闸。
- **为什么能进 🟢 而不是 🟡**：机制**全在编排/agent 层**（脚本解析→排序→记忆→取参考→校验），**不碰扩散模型内部/采样过程**，Nomi 走 vendor API 也能实现；且 Nomi **已有**依赖波次、参考解析、镜级 verify、有界重试四块基建，这篇是把它们用「实体依赖」重新串一遍，不是从零。
- **诚实边界**：① **无开源代码**，只能照正文逻辑重写，不能 clone；② 它带自家 GroundBench，其「参考源镜预测」准不准是命门（预测错→排序错→参考污染），Nomi 落地要用现有 shotVerify 三轴当兜底闸，别让脏参考进记忆；③ 摘要说 model-agnostic 但没点名后端，Nomi 的 Ref2V 能力取决于所接 vendor 是否支持参考图（Nomi 已有 `supportsReferenceImages` 能力位，记忆 `generic-relay-image-edit-and-resolution`），不支持参考图的模型只能退化到 T2V + prompt，效果打折——**这点要在能力契约里 gate 住，别对不支持参考的模型硬上**（P4/防呆）。

## 🟡 架构教训（影响 Nomi 设计决策，非即插即用）

### CoTriSyGen（Closed-Loop Triplet Synergistic）：把「意图/记忆/画面」三元组用 VLM analyzer 闭环，intra-shot 触发重生 + inter-shot 传播新实体（2026-06-15）
[arxiv 2606.16184](https://arxiv.org/abs/2606.16184) · **本轮核实无开源代码**（仅 arxiv）· 作者 Xinlei Yin / Xiulian Peng / Xiao Li / Zhiwei Xiong / Yan Lu（Peng/Lu 常见于 MSRA，未在页面确认）· 摘要读为 inference-time 编排（无训练描述）· 配自家 StoryBench · 日期：2026-06-15。
- **和上一轮 HollyWood Town/MUSE 同族、更细**：三元组 = **planned intent（prompt）× persistent memory × generated visuals**，一个 VLM analyzer 沿两条路回灌：**intra-shot refinement**（检测到语义/构图违规→触发该镜定向重生 + 精化 i2v prompt）、**inter-shot refinement**（把新出现的实体/属性回写进后续镜 prompt）。核心是 **"an entity-centric memory modeled as a mutable visual state that evolves as the story progresses"**。
- **影响 Nomi 哪个决策**：印证 GroundShot 的「entity-centric 可变记忆」不是孤例——**两篇独立工作（GroundShot 训练-free 编排 + CoTriSyGen VLM 闭环）本轮同时收敛到「实体级记忆当可变视觉状态、生成后校验回填」**。给 Nomi 的信号：上面 🟢 那条不是赌单篇，是学界本月的共识方向。CoTriSyGen 多给的一点是「inter-shot 把新实体回写进后续 prompt」——对应 Nomi 拆镜头后「前面镜生出的新造型该同步进后面镜的 prompt/参考」，可作为 🟢 落地时的第二阶段（先做调度+记忆，再做 inter-shot 属性传播）。
- **诚实边界**：无代码、lab 未确认、StoryBench 细节未在摘要给全，当设计参照不当落地源。

### A2RD（Agentic Autoregressive Diffusion）：Retrieve–Synthesize–Refine–Update 逐段闭环，test-time self-improvement，量化转场提升（2026-05-07）
[arxiv 2605.06924](https://arxiv.org/abs/2605.06924) · [项目页 dxlong2000.github.io/AARD](https://dxlong2000.github.io/AARD) · **本轮核实无开源代码**（仅项目页）· 作者 Do Xuan Long / Yale Song / Min-Yen Kan / Tomas Pfister / Long T. Le（含 Google 系 Yale Song/Tomas Pfister，NUS Min-Yen Kan）· **"Hierarchical Test-Time Self-Improvement"→ 推理期自改，非重训**（页面未 100% 明说 training-free）· 日期：2026-05-07。
- **机制 = 逐段闭环**：解耦「创意合成」与「一致性强制」，把长视频当 **Retrieve–Synthesize–Refine–Update** 逐段自改过程；三件套 = Multimodal Video Memory + Adaptive Segment Generation + Hierarchical Test-Time Self-Improvement。摘要报 **"outperforms SOTA by up to 30% in consistency and 20% in narrative coherence"**，人评 **"notable improvements in motion and transition smoothness"**。
- **影响 Nomi 哪个决策**：又一篇「记忆 + 逐段 refine + 闭环」印证（本轮第三篇同向：GroundShot / CoTriSyGen / A2RD）。它的增量是**把 self-improvement 放在 test-time 分层做**，且**量化了转场平滑度提升**——给 Nomi 一个信号：镜级 refine 该分层（先便宜的一致性检查、再贵的重生），和 Nomi 记忆 `pose-self-correcting-loop-and-runtime-audit`「免费先行/VLM 判残差」的分层验证哲学一致。可作 🟢 落地时「verify→repair 该分几层」的参照。
- **诚实边界**：无代码、后端未指定、是否完全 training-free 页面未确认；当分层校验的设计参照。

### BachVid：training-free、**无需参考图**的一致性生成（先生成 identity video → 缓存 DiT 中间量 → 注入新视频）（2025-10-24）
[arxiv 2510.21696](https://arxiv.org/abs/2510.21696) · [项目页 wolfball.github.io/bachvid](https://wolfball.github.io/bachvid) · 作者 Han Yan / Xibin Song / Yifu Wang / Hongdong Li / Pan Ji / Chao Ma · **training-free 但碰 DiT 内部（attention + 中间特征缓存注入）** · 日期：2025-10-24（**踩窗边界，略早于严格 6 月线，仅作教训收**）。
- **机制**：先生成一条 identity video，denoise 时缓存中间变量（用来抽前景 mask + 匹配点），再把缓存注入新视频对应位置，同时锁前景+背景一致，**全程不需要参考图**。
- **为什么只进 🟡**：核心「缓存并注入 DiT 中间量」**必须能碰模型内部采样**，Nomi 走 vendor 黑盒 API **拿不到中间层**，drop-in 不了。价值在**思想**：一致性可以「先立一条基准 identity video、后续镜与它对齐」，而非只靠外部参考图——这和 GroundShot「先生成参考源镜」是同一直觉的不同实现（一个在编排层、一个在采样层），Nomi 只能走编排层那条（GroundShot），采样层这条（BachVid）当反面确认「凡碰模型内部的一致性方案，Nomi 都接不了、别惦记」。
- **诚实边界**：出窗边界日期、碰模型内部、后端仅泛指 DiT；纯当「为什么 Nomi 只能走编排层」的佐证。

### 运镜方向（QWERTY / h-control）：本轮两篇 training-free 相机/运动控制**均碰采样器内部**，Nomi 仍接不了（2026-07 / 2026-05）
[QWERTY 2607.01869](https://arxiv.org/abs/2607.01869)（2026-07-02，ECCV'26，training-free 但 **"manipulate the 3D full attention of DiTs by warping the … queries"**，碰 attention+latent 优化）· [h-control 2605.11871](https://arxiv.org/abs/2605.11871)（2026-05，training-free 但对采样做 block-conditional pseudo-Gibbs 内环精化，碰去噪过程）。

- **诚实边界**：两篇机制真实且新，但与 Nomi 技术栈根本不兼容，纯排除性收录。

[arxiv 2606.13768](https://arxiv.org/abs/2606.13768) · [项目页 snap-research.github.io/CineOrchestra](https://snap-research.github.io/CineOrchestra)（Snap Research）· **需训练（unified video diffusion model）** · 日期：2026-06-11（v2 06-16）。

## 🔵 对标基准（接进 eval 体系自评 Nomi）

### KathaTrace：generator-agnostic 诊断「镜间转场语义坍塌」，给出 Semantic Trajectory Gap 指标 + Semantic Compass 修复/选优探针（2026-07-01，两周新）
[arxiv 2607.01312](https://arxiv.org/abs/2607.01312) · **本轮核实无开源代码 repo**（协议/指标定义在摘要给得较全）· 作者 Jamuna S. Murthy / Amin Karimi Monsefi / Rajiv Ramnath · CC BY 4.0 · 日期：2026-07-01（窗内最新）。
- **它量的正是 DirectorBench 指认的全行业瓶颈**：一种「画面各自看着都一致、但**镜与镜之间「为什么这镜接那镜」的语义没了**」的失败模式（"the loss of transition meaning needed to understand how one scene follows another"）。这就是 Nomi 拆镜头产物最容易翻车的地方——每镜单看没问题，连起来叙事断裂。
- **指标 = Semantic Trajectory Gap（STG）**：在 text-only / image-only / text+image 三种证据下测转场可推断度，**STG = text-only 可推断度 − image-only 可推断度**，量「**视觉化过程中丢了多少转场含义**」。配 KathaBench-25K（5,000 叙事 / 20,000 转场 / 28,712 recoverability 问题），人评 Fleiss' κ=0.845，报了个 substantial STG = 23.5±1.3（即现有生成器普遍丢大量转场语义）。附 **Semantic Compass** 用这些信号做**生成后修复 + storyboard 选优**。
- **怎么接进 eval 体系**：Nomi eval 体系（记忆 `eval-system-plan-status`）目前在「转场/叙事连贯」这条 lane 缺量化尺子（前轮靠 DirectorBench 定性指认、MuSS/EntityBench 偏身份/实体）。STG 给了**可复现、generator-agnostic、只需对生成结果打分**的转场语义指标：对 Nomi 拆镜头产出的 shot 序列，用「文字描述能推断转场 vs 只看画面能推断转场」的差值，量化 Nomi 生成把多少「为什么这么接」的意图弄丢了。落点 = `evals/` lane 新增一条「转场语义损失」度量，喂拆镜头闭环（记忆 `storyboard-closed-loop-verify` 的 shotVerify）当第四轴（现有三轴之外补「镜间转场语义」轴）。**Semantic Compass 的「storyboard 选优」思路**还能反哺 GroundShot 🟢 落地：多候选镜里选转场语义损失最小的那条。
- **诚实边界**：① 无开源代码，STG 要照定义自实现（三证据可推断度需一个 VLM judge，接 Nomi 现有 judge 链路，注意 judge 校准，记忆 `self-improving-harness-loop` 铁律）；② KathaBench-25K 数据未必公开，Nomi 可用自己的拆镜头样本算 STG，不依赖它的数据集；③ 是诊断/指标不是生成方法，只进 eval lane 不产生成能力。
