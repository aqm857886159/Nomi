## 🟢 可落地（这月能进 Nomi）

### TARA（One Rewrite to Fix Them All?）：prompt 修复的「类型化分派 + accept-or-revert 修复闸」——training-free、frozen generator，正好套在 Nomi shotVerify 闭环回灌的改写那一步（2026-07-21，三天新）

[arxiv 2607.18724](https://arxiv.org/abs/2607.18724) · 作者 Haoyue Liu / Xiaoyu Ma / Ye Chen / Shuguang Cui / Xiaoying Tang（Shuguang Cui + Xiaoying Tang 按作者群判港中深系，页面未列 affiliation 未逐一确认）· 本轮**未核实到公开代码仓**（三天新；training-free 编排配方，照思想自实现即可，同 ReBind 处理方式）· 日期：2026-07-21。

- **它点破的失败模式 Nomi 闭环正在犯**：T2I 生成不忠实（数量错/属性张冠李戴/关系含糊/文字不可读）时，现有 prompt 优化器「把异构失败吸进一次统一的 prompt 扩写」——但**每类失败需要的修复语言不同**。Nomi 的闭环回灌现在也是这个形状：verify 检出偏差后，回灌给 agent 的是笼统的「这镜有偏差、请改分镜重生」，没有按偏差类型分派修复话术。
- **TARA 四件套（全在编排层，training-free，四个 frozen generator 上验证）**：① **diagnosis**——把 prompt 拆成原子命题、逐条判失败；② **allocation**——每条失败命题路由给**类型条件化的修复算子**（数量错→数量修复语言，属性错→属性绑定语言…）；③ **compilation**——局部约束编译回一条可执行 prompt；④ **semantic repair gate**——**每次只执行一个指定修复的 accept-or-revert 控制器**，修复后语义没变好就回退，防「修 A 坏 B」的语义回归。DSG/TIFA 双榜 × 4 生成器全部第一（超 VisualPrompter +5.6/+2.6），且最快（16.0s vs 20.0s / prompt）。
- **Nomi 的对应件与缺口（grep 核实）**：
  - diagnosis 层**现成**：`src/workbench/generationCanvas/agent/shotVerify.ts` 三轴（identity/composition/continuity）各带 1-5 档锚点、低于 3 判偏差——**这就是 TARA 的 typed diagnosis**，Nomi 甚至比它多了带档位的量表。
  - governor **现成**：`src/workbench/generationCanvas/agent/storyboardLoopBudget.ts` `decideNext`（done/replan/exhausted）+ 默认封顶 2 轮、绝不静默续花。
  - **缺的是 allocation + gate**（主改动）：「让 AI 修」链路（`src/workbench/generationCanvas/agent/shotVerifyStore.ts` 接偏差卡 → agent 改写）的回灌改写指令模板升级三件：
    1. **按轴分派修复语言**：identity 偏差 → 强化锚定语（`@imageN` 显式归属 + 重述锚节点外观关键词，与 ReBind 落地同底座）；composition 偏差 → 机位/景别/站位写成显式可算词（能从 3D 导演台取的取数值，呼应 NEWTON「规格瓶颈」）；continuity 偏差 → 补前镜衔接约束句（光线/时间/场景延续）。
    2. **atomic repair**：每轮只改失败轴对应的 prompt 片段，pass 的轴对应片段**逐字不动**——防统一重写把已经对的部分改坏（TARA 实测的语义回归就从这来）。
    3. **修复闸**：重生后 re-verify 本就是链路默认，加一条「per-axis 分数对比：失败轴分数没升（或总体降）→ 回退上一版 prompt、轮次照记入 loopBudget」——Nomi 的 1-5 档量表正好当闸的判据，不用新造指标。
- **为什么进 🟢**：整套在编排层、training-free、model-agnostic（frozen generator 前提与 Nomi 走 vendor API 完全同构）；Nomi 三件对应件两件现成，改动集中在一个 prompt 模板 + 一段分数对比逻辑，不是造新系统。
- **诚实边界**：① 论文只在 **T2I** 上验证——Nomi image-first 的**首帧图修复直接适用**；视频镜的 prompt 修复按同思想做但属外推，效果要自己 A/B（别声称论文验过视频）；② 「回退上一版」若要连**产物**一起回退，需节点 result 保留上一版引用——Nomi 节点 result 是否覆写无历史**待核实**，若无历史则先只回退 prompt、产物保留较优那版；③ 无公开代码，数字（+5.6/+2.6）是它自家 matched local setting 实测，Nomi 只吃方法结构不引用数字背书；④ 它每轮只修一个命题、靠多轮迭代收敛——Nomi loopBudget 封顶 2 轮，意味着每轮该修「失败轴里最重的那一条」而不是全修，优先级排序（identity > composition > continuity，身份是命门）要写进模板。

## 🟡 架构教训（影响 Nomi 设计决策，非即插即用）

### Vera：多人身份绑定的失败模式被正式定义成任务（subject confusion / attribute swapping / 过度复制参考图）——身份能力继续下沉进模型内部，Nomi 编排层归属是落地前的桥、落地后的接口（2026-07-22，两天新）

[arxiv 2607.20247](https://arxiv.org/abs/2607.20247) · 作者 Yulong Xu / Xinyue Liu / Shujuan Li / Huafeng Shi / Yan Zhou / Jiwen Liu / Xintao Wang / Yu Shen Liu / Huaibo Huang（同名作者多，lab 归属本轮 WebSearch 未确认，不妄断快手系）· 未见项目页/代码 · 日期：2026-07-22。

- **它把 Nomi 命门的失败模式写成了正式任务定义**：human-centric S2V 里「视频整体看着一致、但身份关键细节仍随帧/姿势/交互漂移」；多人场景下**错误的 identity-role binding** 导致三种失败——subject confusion（认错人）、attribute swapping（属性互换=张冠李戴）、**excessive copying of reference-specific appearance cues（把参考图特有的姿势/表情/穿着整张抄进画面）**。前两种正是 ReBind/MultiRef-Compass 已覆盖的绑定问题；**第三种「过度复制」是 Nomi eval 目前没有的失败轴**——参考图该锁「这个人长什么样」，不该把参考图里的姿势/衣服硬搬进每一镜。
- **解法本体接不了**：百万级 identity-aligned 数据（person-level cross-clip retrieval）+ IFMS（身份聚焦掩码监督）+ RALA（DiT 内 reference-aware 逐层注意力）——训练进模型内部，Nomi 走 vendor 黑盒一律接不了。
- **影响 Nomi 哪个决策**：① **给参考一致性 eval lane 记一条候补轴**：接 MultiRef-Compass（上轮 🔵）时，除「参考保真 + 绑定对不对」外加「**过度复制**」轴——判「成片是否把参考图的 pose/服装在不该复用的镜里硬搬」（VLM judge 可判，成本在 rubric 不在训练）；这也是用户已踩过的真实现象（图生图「不按原图」与「太按原图」是同一根轴的两端）。② **P4 能力位再添一证**：Vera 这类工作产品化后，vendor S2V 会原生带多人绑定——但接口上仍要告诉模型「谁是谁」（Vera 自己也靠 identity correspondence 数据教绑定），所以 **Nomi `@imageN` 显式归属不会因模型变强而白做，它就是那个接口层**；拆镜头执行器该继续按「归属信息结构化、可整段委托给更强模型」设计，别把归属散写在自由文本里。
- **同线信号（并条，不单列）**：[Self Gradient Forcing（arxiv 2607.20368，2026-07-22）](https://arxiv.org/abs/2607.20368)，HF daily papers 当日最热（68 赞，有项目页承诺放码）——两遍训练让未来帧的 loss 反过来监督「历史怎么写进 causal memory」，5 秒训练窗外推到分钟级，**提升点名 subject identity / background-layout consistency**。与 SlotMem（上轮，character-addressable 内存）同属「长时身份靠模型内部记忆解」路线，且是热度最高的一篇。**合读结论**：学界/业界正把跨镜身份从「外部参考图条件」往「模型内部记忆」迁移；Nomi 的编排层身份锁是迁移完成前的桥，迁移完成后 Nomi 的价值位在「身份资产管理 + 归属接口 + 验收」（谁是谁、锁没锁住、漂没漂），不在复刻内存机制。
- **诚实边界**：Vera 无代码/项目页可核，「SOTA」是自家实验口径；「过度复制」轴的具体 rubric 论文没给可抄的定义，接 eval 时要自写锚点并按 JudgeFit 结论做一次标注对齐。
