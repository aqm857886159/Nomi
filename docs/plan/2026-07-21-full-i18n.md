# 全量国际化收口（zh-CN / en）

日期：2026-07-21

## 目标

- 将 `src/` 与 Electron 用户界面中仍可触达的硬编码文案全部迁入统一 i18n 资源。
- 覆盖 JSX 正文/属性之外的数据驱动标签、状态文案、toast/dialog、错误映射、节点元数据、浏览器与时间轴运行时文案。
- 保持默认 `zh-CN`、支持 `zh-CN` / `en`，不改变现有交互、布局、模型协议或用户内容。
- 将国际化门禁从“仅拦新增 JSX 文案”扩展到能发现数据对象与常见运行时出口，最终要求扫描结果归零。

## 范围

- `src/i18n/**`：补齐中英文资源与类型。
- `src/**/*.ts(x)`：迁移所有用户可见硬编码文案；React 外代码统一调用同一 `i18n` 实例。
- `electron/**`：仅迁移 Electron 菜单、对话框、通知等桌面端可见文案；协议字段、日志、模型模板与错误原文不翻译。
- `scripts/check-i18n-visible-text.mjs`：补强静态检查并移除遗留基线依赖。

## 不动项

- AI system prompt、技能/创作模板、模型提示词、供应商请求/响应字段和模型原始名称。
- 用户输入、项目已有内容、网页抓取内容、服务商返回的原始错误详情。
- 当前工作树中与本任务无关的既有改动与生成 CSS。

## 实施策略

1. 先以现有 AST 门禁、中文字符串清单和常见显示字段清单建立完整盘点。
2. 优先复用现有翻译 key；缺失项按功能域补入拆分 locale 文件，避免扩大 `resources.ts`。
3. React 组件使用 `useTranslation()`；非 React 模块使用 `i18n.t()`，需要随语言切换实时变化的数据改为函数或 hook 派生。
4. 增强门禁覆盖对象属性、变量/返回值与桌面端可见出口，并用明确豁免排除协议/提示词等非 UI 内容。
5. 运行零遗留扫描、类型检查、测试、构建，并对中英文主路径做真实界面走查。

## 回滚

- 文案迁移按功能域独立，可逐文件还原；不引入新依赖、不修改持久化结构。
- 门禁增强若出现误报，优先收窄规则或添加带原因的路径/字段豁免，不恢复宽泛遗留基线。

## 验收

1. 中文与 English 下应用壳、项目库、创作区、生成画布、预览/时间轴、模型接入、素材/提示词/技能库、内置浏览器均无固定中文残留或裸 key。
2. toast、确认框、空态、错误卡片、节点状态和数据驱动下拉项随语言切换。
3. `node scripts/check-i18n-visible-text.mjs --require-zero` 通过。
4. `check:filesize`、`check:tokens`、`check:i18n`、`lint:ci`、`typecheck`、`test`、`build` 通过。
